# Fix Issue 使用示例

三个场景覆盖不同调试模式：有明确报错、间歇性故障、无报错但行为异常。

---

## 示例 1：有明确报错 — Cookie 过期导致 PDF 空白

**场景特征**：报错信息清晰，可稳定复现，根因定位快。

**用户**："PDF 生成出来是空白的，报错 `Cookie invalid: session expired`"

### Phase 1 输出

```
✅ 已复现：使用过期 Cookie 运行 `python3 main.py <url> --use-config`，
   生成的 PDF 文件 0 字节。

根因：src/core/auth.py 中 Cookie 过期后未触发刷新/重登录流程，
     直接使用失效凭证请求页面，服务端返回空响应。

计划修改：
| 文件 | 理由 |
|------|------|
| src/core/auth.py | 增加 Cookie 有效性检测，过期时自动引导重登录 |
| config/config.py | 清除无效凭证，避免下次启动仍用过期 Cookie |
```

### Phase 2 → 呈现 AC，等待用户确认

### Phase 3 变更摘要

```
| 文件 | 类型 | 说明 |
|------|------|------|
| src/core/auth.py | 修改 | Cookie 使用前增加过期检测，失效时调用 browser_login() |
| config/config.py | 修改 | 写入新 Cookie 前清除旧缓存 |

根本原因：Cookie 过期后未触发刷新流程
修复方式：在 fetch 前校验 Cookie 有效性，无效则重新登录
风险评估：低 — 只在 Cookie 失效路径增加检测，正常路径无影响
```

### Phase 4 → 运行验证，全部 ✅

### Phase 5 → 更新 CLAUDE.md

## 示例 2：间歇性故障 — 批量下载随机中断

**场景特征**：不能稳定复现，每次中断位置不同，需要放大问题来定位。

**用户**："批量下载到一半就中断了，每次断的位置不一样"

### Phase 1 输出

```
⚠️ 无法稳定复现：运行 3 次批量下载（10 篇），2 次成功，1 次在第 7 篇中断。
   报错：ConnectionError: HTTPSConnectionPool - Read timed out

根因假设（需验证）：
  ① 请求频率过高触发服务端限流（最可能）
  ② 网络抖动导致单次超时后未重试

验证方式：并发压测放大问题
  for i in $(seq 5); do python3 main.py <url> --use-config & done; wait

计划修改：
| 文件 | 理由 |
|------|------|
| src/core/fetcher.py | 增加指数退避重试 + 请求间隔 |
```

### Phase 2 → 呈现 AC，重点关注

```
✅ 必须出现
- [ ] 连续批量下载 10 篇，全部成功，无中断
- [ ] 单次失败自动重试（最多 3 次），超出后抛出明确错误
- [ ] 重试间隔和最大次数可通过配置调整
```

### Phase 3-5 → 按流程执行

## 示例 3：无报错但行为异常 — 内容截断

**场景特征**：没有错误信息，程序"正常"结束，但输出不符合预期。需要逐步排查数据流。

**用户**："PDF 生成了，但只有文章前半部分，后面的代码块和图片都没有"

### Phase 1 输出

```
✅ 已复现：使用 `python3 main.py <url> --use-config` 生成 PDF，
   文件 120KB（正常应 > 500KB），内容在第 3 个代码块后截断。

根因：src/core/converter.py 中滚动加载逻辑只等待了首屏内容。
     极客时间使用 simplebar-content-wrapper 作为滚动容器，
     需要在该容器内持续滚动直到触底，才能加载完整内容。
     当前实现只滚动了外层 window，未触及 simplebar 内部容器。

计划修改：
| 文件 | 理由 |
|------|------|
| src/core/converter.py | 修改滚动目标为 simplebar-content-wrapper |
| config/selectors.json | 确认滚动容器选择器正确 |
```

### Phase 2 → AC 重点

```
✅ 必须出现
- [ ] PDF 文件大小 > 500KB
- [ ] 内容包含所有代码块（对比原文数量）
- [ ] 内容包含所有图片占位（或实际图片）
- [ ] 文章末尾"评论区"元素可见（证明已滚动到底）
```

### Phase 3-5 → 按流程执行
