# 验收标准模板（按 Bug 类型）

替换 `{花括号}` 即可使用。错误标识建议格式：`[模块缩写-编号]`，如 `[AUTH-002]`。

---

## 认证类 — Auth

### Token / Session 失效

```
Bug：[AUTH-002] {Token/Session} 失效后请求未自动刷新

❌ 必须消失
- [ ] 调用受保护接口时抛出 401 / {项目认证错误码}
- [ ] 失效后重新登录，仍反复触发认证错误

✅ 必须出现
- [ ] Token 失效时自动触发刷新流程或引导用户重新登录
- [ ] 刷新成功后新凭证写入持久化存储
- [ ] 后续请求使用新凭证正常返回 2xx

🔬 验证方案
| 单元测试 | {test_auth 路径} -v                    |
| 手动验证 | 清除本地凭证 → 触发请求 → 确认自动刷新/重登录 |
```

### 登录超时

```
Bug：[AUTH-003] 登录流程等待超时

❌ 必须消失
- [ ] 等待用户操作时抛出登录超时异常

✅ 必须出现
- [ ] 超时时间可通过配置调整，不硬编码
- [ ] 超时后给出明确提示，不静默崩溃

🔬 验证方案
| 单元测试 | {test_auth 路径}::test_login_timeout -v |
```

## 网络类 — Fetch / HTTP

```
Bug：[FETCH-001] 批量请求触发限流 / 间歇性超时

❌ 必须消失
- [ ] 批量操作出现 429 / ConnectionError 导致任务中断

✅ 必须出现
- [ ] 加入指数退避重试后，批量任务全部完成
- [ ] 单次失败自动重试；超出最大重试次数后抛出明确错误（含请求信息）
- [ ] 重试次数和间隔可通过配置调整

🔬 验证方案
| 单元测试 | {test_fetcher 路径} -v                   |
| 手动验证 | 批量触发（≥5 次）→ 全部成功，无中断      |
```

## 数据处理类 — Parsing / Conversion

```
Bug：[CONV-001] 输出内容缺失 / 格式错误

❌ 必须消失
- [ ] 输出文件大小异常（如 < 预期阈值）
- [ ] 输出中缺少关键内容（代码块 / 图片 / 正文）

✅ 必须出现
- [ ] 输出包含完整预期内容
- [ ] 输出文件大小在合理范围（> {阈值}）
- [ ] 动态内容完全加载后再处理

🔬 验证方案
| 单元测试 | {test_converter 路径} -v               |
| 手动验证 | 使用典型样本触发 → 检查输出完整性       |

⚠️ 关键检查点
- 异步/懒加载内容是否等待完毕
- 分页 / 滚动触发的内容是否全部获取
- 外部资源（图片/脚本）加载超时是否处理
```

## 配置类 — Config

```
Bug：[CONFIG-001] 配置加载失败 / 解密错误

❌ 必须消失
- [ ] 启动时抛出配置相关异常

✅ 必须出现
- [ ] 配置正确加载并返回预期结构
- [ ] 加载失败时给出明确提示和修复建议（如"请重新配置"）

🔬 验证方案
| 单元测试 | {test_config 路径} -v                   |
| 手动验证 | 删除配置文件 → 启动 → 确认友好提示      |
```

## 环境依赖类 — Environment

```
Bug：[ENV-001] 依赖缺失 / 版本不兼容

❌ 必须消失
- [ ] ImportError / ModuleNotFoundError / 版本冲突报错

✅ 必须出现
- [ ] 所有依赖正确安装，import 无报错
- [ ] 安装命令写入 README / Makefile

🔬 验证方案
| 命令 | {包管理器} install && {smoke test 命令} |
```

## 前端 / 选择器类

```
Bug：前端更新后选择器失配，输出包含噪音元素

❌ 必须消失
- [ ] 输出中出现不应存在的 UI 元素（导航栏、弹层等）
- [ ] 等待目标容器时 TimeoutError

✅ 必须出现
- [ ] 输出只包含目标内容，无多余 UI 元素
- [ ] 配置文件中所有选择器在当前 DOM 中可命中

🔬 验证方案
| 手动定位 | DevTools 控制台验证选择器是否命中目标元素 |
| 手动验证 | 生成输出 → 肉眼检查无噪音元素            |

⚠️ 选择器只能在配置文件中修改，禁止硬编码进业务代码
```

## 并发类 — Race Condition

```
Bug：[CONCUR-001] 并发下结果不一致 / 数据竞争

❌ 必须消失
- [ ] 并发执行时出现错误、丢失或重复数据

✅ 必须出现
- [ ] N 次并发执行，结果全部正确且唯一

🔬 验证方案
| 压测 | 并发 N 次触发 → 验证全部成功（见 04-anti-patterns.md） |
```

## 通用验证命令速查

```bash
# Python
pytest tests/unit/ -x -q          # 快速（失败即停）
pytest tests/ -v                   # 完整套件
mypy src/ --ignore-missing-imports # 类型检查（无测试时降级用）

# Node / TypeScript
npm test -- --passWithNoTests
tsc --noEmit                       # 类型检查降级

# Go
go test ./...
go vet ./...

# 通用冒烟
{启动命令} --help                  # 至少确保能启动
```
