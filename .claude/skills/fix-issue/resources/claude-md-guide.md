# CLAUDE.md 写作规范

> 文件应像代码一样被重构、审查、持续改进。
> 不是写流水账，是把吃过的亏转化为可执行的约束。

---

## 标准条目格式

```markdown
### [YYYY-MM-DD] {问题简述，10字以内}
- **现象：** {用户看到的症状，一句话}
- **根因：** {技术原因，一句话}
- **修复：** {改了什么，为什么有效}
- **防范：** {以后如何避免——必须可执行}
- **相关文件：** {文件路径列表}
- **回归测试：** {测试文件路径}::{测试用例名}
```

---

## 真实填写示例

```markdown
### [2026-04-03] 写缓存后旧值仍被读取
- **现象：** 更新配置后立即读取，返回旧值
- **根因：** write() 写入持久化后未清除内存缓存，下次读取命中旧缓存
- **修复：** write() 后增加 self._cache = None
- **防范：** 所有写入操作后必须同步清除对应内存缓存；新增缓存字段时在此登记
- **相关文件：** src/core/config_manager.py
- **回归测试：** tests/unit/test_config.py::test_cache_invalidated_after_write

### [2026-04-03] 外部依赖更新后选择器失配
- **现象：** 输出中包含不应出现的 UI 元素
- **根因：** 第三方页面 DOM 结构变更，selectors.json 未同步更新
- **修复：** 更新 config/selectors.json 中对应选择器
- **防范：** 依赖方更新时必须检查并更新选择器配置；禁止在业务代码中硬编码选择器
- **相关文件：** config/selectors.json
- **回归测试：** 手动验证输出无噪音元素（暂无自动化用例）
```

## 好的防范措施

防范措施必须**可执行**，不能泛泛而谈。

| ❌ 没用 | ✅ 有用 |
|--------|--------|
| "注意空值处理" | "所有 API 响应解析使用 `.get('key', default)`，禁止直接 `dict['key']`" |
| "小心外部依赖变更" | "第三方更新后必须检查配置文件，禁止硬编码外部结构进业务代码" |
| "做好测试" | "每修复一个 Auth Bug，必须在 test_auth.py 新增对应回归用例" |
| "注意版本升级" | "升级 {库名} 后立即运行冒烟测试并更新 CHANGELOG" |

## 何时必须写（触发条件）

| 触发条件 | 原因 |
|---------|------|
| Bug 涉及非显而易见的初始化顺序或缓存逻辑 | 防止下次同样踩坑 |
| 根因是外部依赖变更（API / DOM / SDK 升级） | 记录兼容性陷阱 |
| 同类问题历史上出现过 2 次以上 | 系统性解决 |
| 修复揭示了隐含的架构约束 | 代码中无法自解释 |
